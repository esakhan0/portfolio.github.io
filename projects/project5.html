Researching the Right Coordinator Software

After purchasing my initial ZigBee devices ‚Äî mainly IKEA products due to their reliability and efficiency ‚Äî I started researching how to coordinate and manage them properly. Most consumer setups rely on manufacturer hubs, but those are often cloud-dependent and restrictive. That‚Äôs when I discovered ZigBee2MQTT. What immediately stood out was its ability to: Run completely locally, Avoid vendor lock-in, Communicate using MQTT (lightweight and fast), Support a wide range of ZigBee devices Integrate easily into containerised environments.For someone building a structured home-lab environment, this was ideal.

I wanted my smart room to operate independently of external servers. Local communication is not only faster, but more reliable and more secure. If my internet goes down, my automations should still work. ZigBee2MQTT made that possible.

Deployment on Docker (Raspberry Pi 5)

Rather than installing ZigBee2MQTT directly onto the OS, I deployed it inside a Docker container running on my Raspberry Pi 5. This was intentional. Running it inside Docker provides: Isolation from the base system, Easy upgrades, Clean configuration management, Quick rebuild if something breaks, Scalability for future services. 

The installation process was straightforward. I pulled the official ZigBee2MQTT image from ghcr.io (GitHub Container Registry), imported it into my Docker server, configured the container settings, and mounted the necessary volumes for persistent data storage. Because my Raspberry Pi 5 already hosts other services, Docker allows me to keep everything modular. Each service runs independently, but they communicate efficiently through MQTT.

The Wireless ZigBee Coordinator (Dongle & Router)

One of the most interesting parts of this project was choosing a wireless ZigBee coordinator. Instead of a standard USB-only dongle that must stay physically attached to the Raspberry Pi, I purchased a Wi-Fi-enabled ZigBee coordinator from Alibaba. This allows the coordinator to connect to my network wirelessly. The benefits of this design: I can position the coordinator anywhere in the house, I can optimise signal placement for better ZigBee mesh coverage, I‚Äôm not limited by USB cable length The Raspberry Pi can stay in my rack/server location. 

After receiving the dongle, the setup process involved: Powering the device, Connecting it to Wi-Fi, Assigning it a stable IP address, Configuring ZigBee2MQTT to point to the coordinator‚Äôs network address. Once connected, ZigBee2MQTT recognised the coordinator immediately. From there, pairing devices became simple. Each IKEA ZigBee device joined the mesh network and began communicating through MQTT.

The ZigBee coordinator handles radio communication. ZigBee2MQTT translates ZigBee signals into MQTT messages.
